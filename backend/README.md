# StudyCompass Backend API

A small, dependency-light REST API for **study materials** (notes, textbooks,
PDFs, slides) — the data layer behind the StudyCompass frontend.

- **Express** (Node.js) with a clean, layered structure
- **File-based JSON store** (`data/materials.json`) — no database to install
- Proper **HTTP status codes**, consistent **JSON** responses
- Central **error-handling** and **404** middleware
- Input **validation** with helpful field-level messages
- **CORS** enabled for the Next.js frontend

## Project structure

```
backend/
├── data/
│   └── materials.json          # file-backed "database" (seeded)
├── postman/
│   └── W4_APICollection_InternID.json   # importable API test collection
├── src/
│   ├── config/env.js           # env loading + defaults
│   ├── controllers/            # request handlers (one per resource)
│   ├── middleware/             # notFound + central errorHandler
│   ├── routes/                 # route definitions
│   ├── services/               # data access (materials.store.js)
│   ├── utils/                  # ApiError, asyncHandler, validate
│   ├── app.js                  # builds the Express app
│   └── server.js               # entry point (starts the server)
├── .env.example                # required env vars (copy to .env)
└── package.json
```

## How to run the backend locally

```bash
# from the repo root
cd backend
npm install          # install dependencies
cp .env.example .env # create your local env file
npm run dev          # start with auto-reload (nodemon)
# or: npm start       # plain node, no reload
```

The API starts on **http://localhost:4000** (configurable via `PORT`).
Verify it's up:

```bash
curl http://localhost:4000/api/health
# { "status": "ok", "service": "studycompass-backend", ... }
```

## Environment variables

Copy `.env.example` to `.env` (which is gitignored) and set:

| Variable      | Default                  | Description                                  |
| ------------- | ------------------------ | -------------------------------------------- |
| `PORT`        | `4000`                   | Port the API listens on                      |
| `NODE_ENV`    | `development`            | `development` exposes error stacks           |
| `CORS_ORIGIN` | `http://localhost:3000…` | Comma-separated list of allowed frontend URLs |

## API reference

Base URL: `http://localhost:4000`

| # | Method   | Endpoint                       | Description                          | Success |
| - | -------- | ------------------------------ | ------------------------------------ | ------- |
| 1 | `GET`    | `/api/health`                  | Liveness check                       | 200     |
| 2 | `GET`    | `/api/materials`               | List all (filters: `?subject=&type=`) | 200     |
| 3 | `GET`    | `/api/materials/search?q=`     | Search title/desc/subject/tags       | 200     |
| 4 | `GET`    | `/api/materials/:id`           | Get one material                     | 200     |
| 5 | `POST`   | `/api/materials`               | Create a material                    | 201     |
| 6 | `PUT`    | `/api/materials/:id`           | Replace a material (all fields)      | 200     |
| 7 | `PATCH`  | `/api/materials/:id`           | Partially update a material          | 200     |
| 8 | `DELETE` | `/api/materials/:id`           | Delete a material                    | 200     |

### Error responses

Errors share one shape and the correct status code:

```jsonc
// 404
{ "error": { "message": "No material found with id 'xyz'", "status": 404 } }

// 400 (validation)
{
  "error": {
    "message": "Validation failed",
    "status": 400,
    "details": { "title": "title is required and must be a non-empty string" }
  }
}
```

### Material schema

```jsonc
{
  "id": "uuid",                  // generated by the server
  "title": "string",            // required
  "subject": "string",          // required
  "type": "note|textbook|pdf|slide", // required
  "description": "string",      // optional
  "tags": ["string"],           // optional
  "pages": 0,                    // optional, non-negative integer
  "createdAt": "ISO date",      // generated
  "updatedAt": "ISO date"       // generated
}
```

## Testing with Postman / Thunder Client

Import `postman/W4_APICollection_InternID.json`. Set the `baseUrl` variable to
`http://localhost:4000` and `materialId` to an existing id. Each request ships
with a saved example response.

## Example requests

```bash
# Create
curl -X POST http://localhost:4000/api/materials \
  -H "Content-Type: application/json" \
  -d '{"title":"Physics Notes","subject":"Physics","type":"note","tags":["mechanics"]}'

# Search
curl "http://localhost:4000/api/materials/search?q=algorithms"

# Partial update
curl -X PATCH http://localhost:4000/api/materials/<id> \
  -H "Content-Type: application/json" -d '{"pages":99}'
```
